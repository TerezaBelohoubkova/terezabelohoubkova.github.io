Case study: AIVA Lab Test results

Simplified user-flow leads to higher user retention

Company

Y Soft

Role

UX designer

Timeline

2 weeks

Tools

Figma

Current UI introduction

tl;dr

My role

During my internship at Y Soft, I got to do a heuristic analysis of an interface AIVA Lab. Not having worked with the interface before, I had a fresh point of view and got to redesign a part of the interface- Test results.

Problem

The AIVA Lab interface used to serve developers, but now new users without technical background are working with it. This was the main reason to optimize the current interface- to serve better to non-tech people.
Users know that their test failed, but don’t know the reason. They really have to put in the work to find out what went wrong. The data they are seeking is buried beneath unimportant data.

Conclusion

The reason of failed robot’s flow should be quickly available to the user. The current interface serves unimportant test data first, important second.
The new design is mainly focused on simplified user flows and lower cognitive load, while staying within an existing design system.

Decision

In order to immediately provide a user with all the important data, I designed a bento grid consisting of tiles summarizing important and frequently used testing data. The solution keeps user’s cognitive load on low and follows Nielsen Norman’s usability heuristics.

Heuristic analysis

In order to determine user problems and pain points, I conducted a heurestic analysis of the interface based on Nielsen Norman Group heurestic principles. The defined problems were based on subjective perception and also user testing done at Y Soft.

Visibility of system status

Problem

When flow has failed, the user doesn't know why. They have to dive deep to find out.

Solution

Add concrete error data and when it occured.

Aesthetic and minimalist design

Problem number 1

User first sees content, that doesn't interest them. They have to click through the important data.

Problem number 2

The bar bellow containing buttons is distracting to the user.

Solution

Design a different layout that prioritizes data based on importance. And change the position of buttons.

Help users recognize, diagnose, and recover from errors

Problem

Error messages in logs are confusing and the user doesn't know what they mean.

Solution

Change language of error messages.

Recognition rather than recall

Problem

When the user wants to play the exact time an error occurred on video, they have to remember what time the error occurred then go to video and find exact time slot.

Solution

Logs should be available at the same time as video.

Match between system and real world

Problem

A filter icon that hides a logs filter. The icon suggests settings rather than a filter.

Solution

Change icon of filtering logs.

User control and freedom

Problem

It isn't clear whether the close button will close only Test results or also whole Reports.

Solution

Change close button to go-back button instead. The user goes back to Reports.

Consistency and standards

Problem

There is information above the video that looks like buttons but isn't clickable.

Solution

Hide unnecessary video information about the recording.

Bento grid sketch

Low fidelity wireframe

A bento grid layout solves the issue of immediate availability of data. This way the user sees everything at once and doesn't have to click through tabs and remember information.
How did I come up with this? I was heavily influenced by design trends of 2024 and been wanting to try out a bento grid design for a long time. This project seemed like the best opportunity.

Fullfiling user needs and accessibility

Iterative design

Based on the low-fi wireframe, I started designing a high fidelity prototype. The first prototype being a minimum viable product. After receiving feedback from other UX designers and checking accessibility of elements, I continued designing until the final design fulfilled all the user and accessibility needs.

Old & New

Comparing current UI with the design

To better understand the changes and the impact the design can potentially have on the business, let's compare the current UI and the design based on user experience.

General test info & occurred errors list

Current UI

There was too much general information that wasn't necessary for the user. It was also taking up a lot of space and the user was first met with the least important information.
Another issue was the Flow Failed status, when the user doesn't immediately understand the reason for failure. The user would have to scroll down and search in logs to understand the moment of failure.

Design

In order to make more important test results info stand out, the general information had to be cut down to 5 most important: flow status, robot name, device, time and flow name. All of these are pinned at the top of the screen and more information is available through the show more icon.
The biggest change was adding a list of occurred errors to the layout. The error messages are rewritten for users to better understand the issues and there's also a time when the error happened. The goal was to make the error information better accessible to the user.

Video & images

Current UI

When wanting to play a video of the robot's flow, the user was met with button-like video data, that didn't interest the user. The video and list of images were burried in tabs. When the user wanted to look at images and video at the same time, they needed to have several browser tabs open.

Design

The list of images and video were brought out of the tabs and are better accessible to the user. There is no unnecessary data about the video. Labels in images are accessible according to the WCAG guidelines.

Logs & measurements

Current UI

A huge problem with logs was a filter and search feature hidden in the settings icon, that is known only to developers. In order to find the occurred errors the user had to scroll down and filter out error logs in order to find out cause of flow failure.
To see the measured data during flow, the user had to open a tab and see it.

Design

The filters are available through the filter icon, while the search bar was moved to a bar above logs to improve accessibility. The UI of measurements was kept the same but was moved to the bento grid layout for better availability.

Learnings

Getting feedback matters

Feedback pushed the boundaries that I would involuntarily set in my head. It allowed me to think more critically and make user-centered decisions.

The first design isn't the final one

The design process is iterative for a reason: to create the best interface that aligns user and stakeholder needs.
To validate that a decision I made is the right one, it's convenient to try out different ideas. That way I get more confidence in the final design.

Following accessibility principles enhances the design

Accessibility is a big factor in UX design, as it ensures the product can be used by anyone. The group of potential users becomes broader, hence higher conversion rates of the product.
